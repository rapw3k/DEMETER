DEMETER Agriculture Information Model (AIM) Ontology Specification Draft

Abstract

The DEMETER Agriculture Information Model (AIM) is the common vocabulary in DEMETER project providing the basis for semantic interoperability across smart farming solutions.

Introduction back to ToC

AIM was originally developed as part of DEMETER project, and is being reused and extended in many ohter projects related to agriculture, but also in other domains (reusing the cross-domain layer). This module is one of the 13 core modules in the DEMETER AIM ontology:
    - DEMETER Agriculture Information Model (AIM) domain ontology
    - AIM FarmAnimal
    - DEMETER core meta-model
    - DEMETER Cross-Domain
    - DEMETER AgriSystem
    - DEMETER AgriProperty
    - DEMETER AgriProduct
    - DEMETER AgriPest
    - DEMETER AgriIntervention
    - DEMETER AgriFeature
    - DEMETER AgriCrop
    - AIM AgriCommon
    - DEMETER AgriAlert

DEMETER AIM ontology: DEMETER core meta-model: Description back to ToC

The DEMETER core ontology includes the meta-model elements reused from NGSI-LD specification that provide a formal basis for representing 'property graphs' using RDF/RDFS/OWL. The DEMETER core ontology is a profile (subset) of the ontology presented in Annex D of the NGSI-LD standard: ETSI GS CIM 006 V1.1.1 (2019-07), covering only the core property-graph entities.
